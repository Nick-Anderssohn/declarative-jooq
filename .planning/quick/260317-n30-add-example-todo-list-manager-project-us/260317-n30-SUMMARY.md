## What Was Built

### Task 1: maven-publish for dsl-runtime and codegen

Added `maven-publish` plugin and `MavenPublication` to `dsl-runtime/build.gradle.kts` and `codegen/build.gradle.kts`. Added `allprojects { version = "0.1.0-SNAPSHOT" }` to root `build.gradle.kts`. All three library modules (dsl-runtime, codegen, gradle-plugin) now publish to `~/.m2/repository/com/nickanderssohn/` via `./gradlew publishToMavenLocal`.

### Task 2: Standalone todo-list example project

Created `examples/todo-list/` as a fully standalone Gradle project (own wrapper, own build, own settings — NOT referenced from root `settings.gradle.kts`).

Key design choices:
- **Hand-written jOOQ classes** in `com.nickanderssohn.todolist.jooq` package — `TodoListTable`, `TodoItemTable` (with FK via `getReferences()`), and their `UpdatableRecordImpl` record classes
- **declarative-jooq plugin** configured with `packageFilter` pointing to the jooq package; `generateDeclarativeJooqDsl` depends on `compileKotlin`, `compileTestKotlin` depends on `generateDeclarativeJooqDsl`
- **Flyway migration** creates `todo_list` and `todo_item` with FK relationship
- **Spring Boot REST API** with jOOQ repositories, returning DTOs (not raw records)
- **Integration tests** use `@SpringBootTest + @Testcontainers + @DynamicPropertySource` with the same Docker detection workarounds as root integration-tests

Integration test covers:
1. `execute(dslContext) { todoList { todoItem { ... } todoItem { ... } } }` — DSL seeding with auto FK wiring
2. REST API CRUD via `TestRestTemplate`
3. Seeded data accessible via REST endpoints
4. 404 for non-existent todo list

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] gradle-plugin missing jOOQ runtime dependency**
- **Found during:** Task 2, running `generateDeclarativeJooqDsl`
- **Issue:** `ClassNotFoundException: org.jooq.impl.TableImpl` — the Gradle plugin task's URLClassLoader uses the Gradle worker's context classloader as parent, which did not have jOOQ on its classpath
- **Fix:** Added `implementation("org.jooq:jooq:3.19.16")` to `gradle-plugin/build.gradle.kts` — this is the first time the plugin has been used in a real project context (integration tests called CodeGenerator directly, not via the Gradle task)
- **Files modified:** `gradle-plugin/build.gradle.kts`
- **Commit:** 68cf1e5

**2. [Rule 1 - Bug] Spring Boot 3.4.x requires JVM 17, not 11**
- **Found during:** Task 2, initial compilation
- **Issue:** Plan specified JVM 11 target but Spring Boot 3.4.3 requires JVM 17+
- **Fix:** Changed `sourceCompatibility`, `targetCompatibility`, and Kotlin `jvmTarget` to 17 in `examples/todo-list/build.gradle.kts`
- **Files modified:** `examples/todo-list/build.gradle.kts`
- **Commit:** 68cf1e5

**3. [Rule 1 - Bug] NOT NULL columns with DB defaults fail when jOOQ builder sets null**
- **Found during:** Task 2, first test run
- **Issue:** Generated builder calls `record.set(field, null)` for all fields; `created_at NOT NULL DEFAULT NOW()` and `completed NOT NULL DEFAULT FALSE` violated because explicit null overrides DB defaults
- **Fix:** Removed `created_at` from table field definitions (DB handles the default via `DEFAULT NOW()`); changed `completed` to nullable in migration (`BOOLEAN DEFAULT FALSE`) since the generated DSL doesn't set it when not specified
- **Files modified:** `examples/todo-list/src/main/kotlin/com/nickanderssohn/todolist/jooq/TodoListTable.kt`, `TodoItemTable.kt`, `TodoListRecord.kt`, `TodoItemRecord.kt`, `V1__create_todo_schema.sql`
- **Commit:** 68cf1e5

**4. [Rule 1 - Bug] jOOQ UpdatableRecordImpl cannot be serialized as JSON by Jackson**
- **Found during:** Task 2, REST API tests
- **Issue:** Controllers returned `TodoListRecord`/`TodoItemRecord` directly; Jackson serializes jOOQ records as arrays, not named JSON objects, causing `MismatchedInputException` in tests
- **Fix:** Created `TodoListDto` and `TodoItemDto` data classes with extension functions `toDto()`; updated controllers to return DTOs
- **Files modified:** Added `examples/todo-list/src/main/kotlin/com/nickanderssohn/todolist/model/TodoListDto.kt`, updated both controllers
- **Commit:** 68cf1e5

**5. [Rule 2 - Missing functionality] DSL extension function requires import in test**
- **Found during:** Task 2, test compilation
- **Issue:** Generated `fun DslScope.todoList(...)` is in `com.nickanderssohn.todolist.generated` package; test needed explicit `import com.nickanderssohn.todolist.generated.todoList`
- **Fix:** Added the import to `TodoListIntegrationTest.kt`
- **Commit:** 68cf1e5
