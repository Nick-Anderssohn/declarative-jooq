IMPORTANT DESIGN NOTE: The builder pattern needs careful thought. Each call to a builder function (e.g., `organization { }`) should:
1. Execute the lambda to set field values and declare children
2. Call `buildRecord()` to create the jOOQ record
3. Create a `RecordNode` and add it to the graph
4. For child builders, pass the parent's RecordNode so FK resolution knows the parent

The recommended pattern:
- The `organization { }` extension function on DslScope creates an OrganizationBuilder, calls the lambda on it, then calls `build()` to get the RecordNode and adds it as a root node
- Inside OrganizationBuilder, `user { }` creates an AppUserBuilder with `this`'s built node as parent, calls the lambda, builds the child node
- The builder must track its own built node so child builders can reference it

Revised approach -- builders should work like this:

```kotlin
class OrganizationBuilder(
    private val recordGraph: RecordGraph
) : RecordBuilder<OrganizationRecord>(
    table = OrganizationTable.ORGANIZATION,
    parentNode = null,
    parentFkField = null,
    recordGraph = recordGraph
) {
    var name: String? = null
    private var builtNode: RecordNode? = null
    private val childBlocks = mutableListOf<() -> Unit>()

    override fun buildRecord(): OrganizationRecord {
        val record = OrganizationRecord(OrganizationTable.ORGANIZATION)
        record.set(OrganizationTable.ORGANIZATION.NAME, name)
        return record
    }

    fun user(block: AppUserBuilder.() -> Unit) {
        childBlocks.add {
            val builder = AppUserBuilder(
                recordGraph = recordGraph,
                parentNode = builtNode!!,
                parentFkField = AppUserTable.APP_USER.ORGANIZATION_ID
            )
            builder.block()
            val childNode = builder.build()
            // childNode is already added to parent's children by RecordBuilder.build()
        }
    }

    fun buildWithChildren(): RecordNode {
        builtNode = build()
        childBlocks.forEach { it() }
        return builtNode!!
    }
}

class AppUserBuilder(
    recordGraph: RecordGraph,
    parentNode: RecordNode,
    parentFkField: org.jooq.TableField<*, *>
) : RecordBuilder<AppUserRecord>(
    table = AppUserTable.APP_USER,
    parentNode = parentNode,
    parentFkField = parentFkField,
    recordGraph = recordGraph
) {
    var name: String? = null
    var email: String? = null

    override fun buildRecord(): AppUserRecord {
        val record = AppUserRecord(AppUserTable.APP_USER)
        record.set(AppUserTable.APP_USER.NAME, name)
        record.set(AppUserTable.APP_USER.EMAIL, email)
        return record
    }
}

// Extension function on DslScope -- this is what generated code would produce
fun DslScope.organization(block: OrganizationBuilder.() -> Unit) {
    val builder = OrganizationBuilder(recordGraph)
    builder.block()
    val node = builder.buildWithChildren()
    recordGraph.addRootNode(node)
}
```
