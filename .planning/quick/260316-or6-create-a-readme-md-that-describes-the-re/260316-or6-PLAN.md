Key project facts for the README:
- Kotlin library wrapping jOOQ for declarative test data creation
- 4 modules: dsl-runtime, codegen, gradle-plugin, integration-tests
- Tech stack: Kotlin 2.1.20, jOOQ 3.19.16, KotlinPoet 2.1.0, Gradle 8.12
- JVM target: 11
- Group ID: com.nickanderssohn
- Gradle plugin ID: com.nickanderssohn.declarative-jooq
- Published via mavenLocal only
- Tests: H2 (unit), Postgres 16 via Testcontainers (integration)

DSL entry point is `execute(dslContext) { ... }` from com.nickanderssohn.declarativejooq.
Code generator scans jOOQ record classes and produces typed DSL builder extension functions.
Gradle plugin registers `generateDeclarativeJooqDsl` task with `declarativeJooq` extension block.

Extension config properties:
- classesDir: Directory containing compiled jOOQ record classes (required)
- outputPackage: Output package for generated DSL code (required)
- packageFilter: Optional package filter to restrict scanned jOOQ packages

Plugin auto-wires generated sources into the test source set.

Example DSL usage (from integration tests):
```kotlin
val result = execute(ctx) {
    organization {
        name = "Acme"
        organization {  // creates app_user nested under org
            name = "Alice"
            email = "alice@acme.com"
            createdBy {  // creates task nested under user via created_by FK
                title = "Alice's Task"
            }
        }
    }
    category {
        name = "Electronics"
        childCategory {  // self-referential FK
            name = "Phones"
        }
    }
}
```

DslResult API:
- `records(tableName)` returns List<UpdatableRecord<*>>
- `allRecords()` returns Map<String, List<UpdatableRecord<*>>>
