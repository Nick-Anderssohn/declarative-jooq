```yaml
name: CI

on:
  pull_request:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '11'
          cache: 'gradle'

      - uses: gradle/actions/setup-gradle@v5

      - name: Build and test
        run: ./gradlew build
        env:

```

Key design points:
- `on: pull_request` (no branch filter = all PRs) + `on: push: branches: [main]`
- `ubuntu-latest` has Docker pre-installed -- no Docker setup step needed for Testcontainers
- `./gradlew build` runs all subproject test tasks including `:integration-tests:test` (verified via `--dry-run`)
- `TESTCONTAINERS_RYUK_DISABLED: "true"` matches the project's local configuration in `integration-tests/build.gradle.kts`
- `actions/setup-java@v4` with `cache: 'gradle'` for dependency caching
- `gradle/actions/setup-gradle@v5` for Gradle home and build cache (replaces deprecated `gradle-build-action`)
- No publishing credentials or tasks -- this workflow only builds and tests
- Java 11 matches project's `JvmTarget.JVM_11` and `sourceCompatibility = "11"`

Create `.github/workflows/publish.yml` with the following exact content:

```yaml
name: Publish

on:
  push:
    tags:
      - 'v*'

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '11'
          cache: 'gradle'

      - uses: gradle/actions/setup-gradle@v5

      - name: Publish to Maven Central
        run: ./gradlew publishToMavenCentral
        env:
          ORG_GRADLE_PROJECT_mavenCentralUsername: ${{ secrets.SONATYPE_USERNAME }}
          ORG_GRADLE_PROJECT_mavenCentralPassword: ${{ secrets.SONATYPE_PASSWORD }}
          ORG_GRADLE_PROJECT_signingInMemoryKey: ${{ secrets.SIGNING_KEY }}
          ORG_GRADLE_PROJECT_signingInMemoryKeyId: ${{ secrets.SIGNING_KEY_ID }}
          ORG_GRADLE_PROJECT_signingInMemoryKeyPassword: ${{ secrets.SIGNING_PASSWORD }}
```

Key design points:
- Trigger is ONLY `on: push: tags: ['v*']` -- NO `branches:` key under `push:`, NO `pull_request:` key
- This ensures the workflow fires only on tag pushes like `v1.0.0`, never on branch pushes or PRs
- Task is `publishToMavenCentral` (NOT `publishAndReleaseToMavenCentral`) because `automaticRelease = false` is set in all three modules -- this uploads and closes the staging bundle but does NOT auto-release
- After this workflow completes, the developer must visit central.sonatype.com to manually inspect and release the bundle
- All 5 credential env vars use the `ORG_GRADLE_PROJECT_` prefix which Gradle automatically maps to project properties:
  - `SONATYPE_USERNAME` -> `ORG_GRADLE_PROJECT_mavenCentralUsername` -> Gradle property `mavenCentralUsername`
  - `SONATYPE_PASSWORD` -> `ORG_GRADLE_PROJECT_mavenCentralPassword` -> Gradle property `mavenCentralPassword`
  - `SIGNING_KEY` -> `ORG_GRADLE_PROJECT_signingInMemoryKey` -> Gradle property `signingInMemoryKey`
  - `SIGNING_KEY_ID` -> `ORG_GRADLE_PROJECT_signingInMemoryKeyId` -> Gradle property `signingInMemoryKeyId`
  - `SIGNING_PASSWORD` -> `ORG_GRADLE_PROJECT_signingInMemoryKeyPassword` -> Gradle property `signingInMemoryKeyPassword`
- Java 11 matches project target
