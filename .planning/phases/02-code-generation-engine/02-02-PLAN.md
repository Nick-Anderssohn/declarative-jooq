// ROOT BUILDER PATTERN â€” OrganizationBuilder
class OrganizationBuilder(private val graph: RecordGraph) : RecordBuilder<OrganizationRecord>(
    table = OrganizationTable.ORGANIZATION,
    parentNode = null,
    parentFkField = null,
    recordGraph = graph
) {
    var name: String? = null
    private val childBlocks = mutableListOf<(RecordNode) -> Unit>()

    override fun buildRecord(): OrganizationRecord {
        val record = OrganizationRecord()
        record.set(OrganizationTable.ORGANIZATION.NAME, name)
        return record
    }

    fun user(block: AppUserBuilder.() -> Unit) {
        childBlocks.add { parentNode ->
            val builder = AppUserBuilder(
                recordGraph = graph,
                parentNode = parentNode,
                parentFkField = AppUserTable.APP_USER.ORGANIZATION_ID
            )
            builder.block()
            builder.build()
        }
    }

    fun buildWithChildren(): RecordNode {
        val node = build()
        childBlocks.forEach { it(node) }
        return node
    }
}

        val record = AppUserRecord()
        record.set(AppUserTable.APP_USER.NAME, name)

    }

1. Create `BuilderEmitter.kt` in `codegen/src/main/kotlin/com/example/declarativejooq/codegen/emitter/`:
   - Class `BuilderEmitter` with method `emit(tableIR: TableIR, outputPackage: String): TypeSpec`
   - Uses KotlinPoet to generate a builder class. Two modes based on whether table has inboundFKs (is a parent):

   **For ROOT builders** (tables where `isRoot == true` OR tables that have `inboundFKs`):
   - Constructor takes `graph: RecordGraph` (private val)
   - Superclass: `RecordBuilder<{RecordClass}>` with args: `table = {TableClass}.{TABLE_CONSTANT}`, `parentNode = null`, `parentFkField = null`, `recordGraph = graph`
   - Mutable `var` properties for each non-identity column (type is `kotlinTypeName.copy(nullable = true)`), initialized to `null`
   - Override `fun buildRecord(): {RecordClass}` â€” creates `val record = {RecordClass}()`, for each non-identity column: `record.set({tableFieldRefExpression}, {propertyName})`, returns record
   - Private `val childBlocks = mutableListOf<(RecordNode) -> Unit>()`
   - For each FK in `inboundFKs`: a child builder function named `{fk.builderFunctionName}` that takes `block: {ChildBuilderClass}.() -> Unit` and adds a lambda to `childBlocks` that creates the child builder with `parentNode = parentNode`, `parentFkField = {fk.childFieldExpression}`, calls `builder.block()`, calls `builder.build()`
   - `fun buildWithChildren(): RecordNode` that calls `val node = build()`, `childBlocks.forEach { it(node) }`, returns `node`

   **For CHILD-ONLY builders** (tables where `isRoot == false` AND `inboundFKs.isEmpty()`):
   - Constructor takes `recordGraph: RecordGraph`, `parentNode: RecordNode`, `parentFkField: org.jooq.TableField<*, *>` â€” NOT private, passed through to super
   - Superclass: `RecordBuilder<{RecordClass}>` with args: `table = {TableClass}.{TABLE_CONSTANT}`, `parentNode = parentNode`, `parentFkField = parentFkField`, `recordGraph = recordGraph`
   - Same mutable var properties and buildRecord() override as root
   - NO childBlocks, NO buildWithChildren(), NO child builder functions

   **For tables that are BOTH parent AND child** (have both outboundFKs and inboundFKs):
   - Constructor takes `recordGraph: RecordGraph`, `parentNode: RecordNode`, `parentFkField: org.jooq.TableField<*, *>` â€” same as child
   - But ALSO has childBlocks, child builder functions for its inboundFKs, and buildWithChildren()
   - This handles intermediate tables in a chain (e.g., department belongs to org, has employees)
   - NOTE: For root extension function emission, only tables with `isRoot == true` get DslScope extension functions. But a non-root table can still be a parent.

   **Important KotlinPoet details:**
   - Use `ClassName(tableIR.sourcePackage, tableIR.recordClassName)` for the record type
   - Use `ClassName(tableIR.sourcePackage, tableIR.tableClassName)` for table class references
   - Use `ClassName("com.example.declarativejooq", "RecordBuilder").parameterizedBy(recordType)` for superclass
   - Use `ClassName("com.example.declarativejooq", "RecordGraph")` for RecordGraph type
   - Use `ClassName("com.example.declarativejooq", "RecordNode")` for RecordNode type
   - For `record.set(TABLE.CONSTANT.FIELD, value)` in buildRecord, emit as `CodeBlock.of("record.set(%L, %L)", col.tableFieldRefExpression, col.propertyName)` â€” the tableFieldRefExpression is a raw code string like `"OrganizationTable.ORGANIZATION.NAME"`
   - For child builder FK field reference in the lambda, use `CodeBlock.of("%L", fk.childFieldExpression)` â€” raw code like `"AppUserTable.APP_USER.ORGANIZATION_ID"`

        - Call `fileSpec.writeTo(outputDir)` to write to disk
